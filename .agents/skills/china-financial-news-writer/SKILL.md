# 中国金融新闻自动写作系统 (China Financial News Writer)

专业级金融新闻自动生成系统，根据公司类型、新闻类型、输出风格智能匹配写作框架，一键生成高质量金融内容。

## 核心特性

- **万能分析框架**: 12大模块覆盖财经分析所有维度，适用于任何公司危机事件
- **三维分类矩阵**: 公司类型 × 新闻类型 × 输出风格 = 定制化写作大纲
- **深度情报搜集**: 全网 6 维调研（新闻/视频/社交/论坛/官方/数据）⭐ NEW
- **智能模块选择**: 根据平台特性和时长要求，自动推荐最优内容组合
- **敏感词合规**: 内置金融领域敏感词检测与替换
- **关键词优化**: 自动布局核心词、长尾词、场景词
- **互动触发设计**: 针对不同平台的互动机制设计
- **数据驱动**: 支持财报数据、市场数据自动整合

## 三维分类矩阵

### 维度一：公司类型 (Company Types)

| 类型 | 代表公司 | 核心关注点 | 写作侧重 |
|------|----------|------------|----------|
| **科技巨头** | 腾讯、阿里、字节、美团、京东 | 用户增长、变现效率、新业务 | 生态协同、技术壁垒、监管影响 |
| **新能源车企** | 比亚迪、蔚来、小鹏、理想、小米汽车 | 销量、交付量、毛利率 | 技术路线、产能扩张、出海战略 |
| **消费品牌** | 安踏、李宁、珀莱雅、完美日记、瑞幸 | 同店增长、渠道扩张、品牌力 | 消费趋势、渠道变革、国潮红利 |
| **金融券商** | 中信、中金、华泰、东方财富 | AUM、交易量、投行业务 | 市场环境、政策变化、财富管理 |

### 维度二：新闻类型 (News Types)

| 类型 | 触发场景 | 分析框架 | 时间敏感度 |
|------|----------|----------|------------|
| **财报分析** | 季报/年报发布 | Beat/Miss 分析、指引解读、趋势判断 | 极高（24h 内） |
| **产品发布** | 新品发布会/公告 | 产品力评估、市场空间、竞争格局 | 高（48h 内） |
| **行业动态** | 行业事件/数据 | 产业链影响、受益标的、风险提示 | 中（72h 内） |
| **政策影响** | 政策发布/解读 | 政策意图、受益/受损方、投资逻辑 | 高（48h 内） |

### 维度三：输出风格 (Output Styles)

| 风格 | 字数 | 表格 | 图表 | 语气 | 目标读者 |
|------|------|------|------|------|----------|
| **小红书** | 500-800 | 0-1 | 3-5 | 亲切/emoji/口语化 | 个人投资者/理财小白 |
| **公众号** | 1500-2500 | 1-3 | 5-8 | 专业但不失活泼 | 泛财经读者/投资爱好者 |
| **研报简报** | 3000-5000 | 3-5 | 8-12 | 专业/客观/数据密集 | 专业投资者/机构客户 |
| **深度报告** | 5000-8000 | 5-8 | 10-15 | 专业/深度/多维度 | 专业投资者/研究员 |

## 写作流程 (7-Phase Workflow)

### Phase 0: 万能分析框架应用

**目标**: 使用12大模块框架全面诊断企业危机事件

**模块选择指南**:

| 平台类型 | 推荐模块组合 | 时长分配 |
|---------|-------------|----------|
| **B站视频** | 1,2,3 + 6,9,10,11 | 事件1min+分析3min+对比2min+技术2min+故事1min+互动1min |
| **小红书** | 1,2,3 + 9,10 | 事件+分析+对比+故事+共鸣 |
| **公众号** | 1,2,3,4,6,7 | 事件+分析+数据+技术+历史+展望 |
| **深度报告** | 全模块 | 按重要性权重分配 |

详见 [references/universal_financial_analysis_framework.md](references/universal_financial_analysis_framework.md)

### Phase 1: 信息采集 (Data Collection)

```
1. 确认新闻源与时效性
   ├── 检查发布日期（需在 72 小时内）
   ├── 标注数据来源（财报/公告/新闻）
   └── 交叉验证关键数据

2. 提取核心数据点
   ├── 财报类：营收、利润、毛利率、指引
   ├── 产品类：参数、价格、发布时间
   ├── 行业类：市场规模、增速、份额
   └── 政策类：核心条款、生效时间

3. 竞品/行业对比数据
   └── 收集对标公司同期数据
```

详见 [references/data-collection.md](references/data-collection.md)

---

### Phase 1.5: 深度情报搜集 (Deep Intelligence Gathering) ⭐ NEW

**目标**：像 ChatGPT 一样全网搜集信息，完整梳理事件的来龙去脉、历史渊源、深层原因

**6 维情报网**：

```
1. 新闻媒体报道
   ├── 主流财经（虎嗅、36 氪、财新、界面）
   ├── 门户财经（新浪、网易、腾讯）
   ├── 垂直媒体（第一财经、证券时报）
   └── 海外媒体（WSJ 中文、FT 中文、Bloomberg）

2. 视频平台内容
   ├── B 站财经 UP 主分析
   ├── 抖音/快手短视频
   └── YouTube 中文财经频道

3. 社交媒体讨论
   ├── 微信公众号（搜狗微信搜索）
   ├── 微博热搜/话题
   ├── 小红书笔记
   └── 豆瓣小组讨论

4. 投资社区/论坛
   ├── 雪球 - 投资者深度分析
   ├── 东方财富股吧 - 散户情绪
   ├── 知乎 - 专业分析回答
   └── 老虎社区/富途牛牛

5. 官方信息源
   ├── 公司官网/投资者关系
   ├── 交易所公告（港交所/上交所/深交所）
   ├── 监管文件
   └── 业绩会录音/纪要

6. 数据工具
   ├── 百度指数/微信指数 - 热度趋势
   ├── 天眼查/企查查 - 关联关系
   └── 行业数据库
```

**输出物**：
- 事件时间线（何时发布、数据如何、历史对比）
- 原因分析框架（销量？成本？汇率？竞争？）
- 全网观点汇总（分析师、媒体、散户情绪）
- 信息可信度评估

**深度报告输出**：
- 使用万能分析框架12大模块
- 字数：5000-8000 字
- 包含：事件引爆点、战略失误分析、市场竞争格局、技术路线分析、历史对比、未来预测

详见 [references/deep-research.md](references/deep-research.md) 和 [references/research-report-template.md](references/research-report-template.md)

### Phase 2: 分析框架 (Analysis Framework)

根据新闻类型选择分析框架：

#### 财报分析 - 8 维度模型

| 维度 | 权重 | 分析要点 |
|------|------|----------|
| **业绩惊喜** | 25% | 实际 vs 预期差异、超预期/不及预期原因 |
| **核心指标** | 20% | 收入增速、利润率、ROE、现金流 |
| **业务结构** | 15% | 分业务/分地区拆解、亮点与隐忧 |
| **指引解读** | 15% | 管理层展望、上调/下调预期 |
| **行业对比** | 10% | vs 竞品表现、行业趋势 |
| **估值水平** | 10% | 当前 PE/PB vs 历史区间 |
| **风险因素** | 5% | 潜在风险点、关注事项 |
| **催化剂** | 5% | 未来股价驱动因素 |

#### 产品发布 - 5 维度模型

```
1. 产品力评估 (30%)
   ├── 核心参数 vs 竞品
   ├── 创新点/差异化
   └── 目标用户画像

2. 市场空间 (25%)
   ├── TAM/SAM/SAM 估算
   ├── 增量 vs 存量市场
   └── 渗透率预期

3. 竞争格局 (20%)
   ├── 主要竞争对手
   ├── 市场份额预期
   └── 护城河分析

4. 商业模式 (15%)
   ├── 定价策略
   ├── 毛利率预期
   └── 销售渠道

5. 时间线 (10%)
   ├── 上市时间
   ├── 产能爬坡
   └── 业绩贡献预期
```

#### 行业动态 - 4 维度模型

```
1. 事件概述 (20%)
   └── 事件背景、关键信息

2. 影响分析 (40%)
   ├── 产业链影响路径
   ├── 受益方分析
   └── 受损方分析

3. 投资逻辑 (30%)
   ├── 短期催化
   ├── 中长期逻辑
   └── 相关标的

4. 风险提示 (10%)
   └── 不确定性因素
```

#### 政策影响 - 4 维度模型

```
1. 政策解读 (30%)
   ├── 核心条款
   ├── 与市场预期的差异
   └── 实施时间表

2. 行业影响 (30%)
   ├── 受益板块
   ├── 受损板块
   └── 结构性机会

3. 投资建议 (30%)
   ├── 短期策略
   ├── 中期布局
   └── 相关标的

4. 风险提示 (10%)
   └── 政策执行不确定性
```

```
1. 敏感词扫描
   ├── 高危词：保本、稳赚、100%、内幕
   ├── 中危词：推荐买入、强烈推荐、确定性强
   └── 替换建议

2. 投资建议合规
   ├── 禁止承诺收益
   ├── 必须提示风险
   └── 免责声明

3. 数据来源标注
   └── 确保所有数据可追溯
```

详见 [references/compliance-rules.md](references/compliance-rules.md)

## 输出规格 (Output Specification)

### 小红书风格

```markdown
**标题**: [情绪词] + [核心结论] + [数字/悬念]
**字数**: 500-800 字
**结构**:
  - 开头：痛点/场景引入（50 字）
  - 核心内容：3-5 个要点（300-500 字）
  - 结尾：互动引导（50 字）
**格式**: emoji 穿插、短段落、口语化
**标签**: 5-10 个
```

### 公众号风格

```markdown
**标题**: [关键词] + [价值点] + [时效性]
**字数**: 1500-2500 字
**结构**:
  - 导语：新闻概述（200 字）
  - 核心分析：3-5 个维度（1000-1500 字）
  - 投资建议/结论（300 字）
  - 风险提示（100 字）
**格式**: 专业但不失活泼、数据 + 观点
**配图**: 5-8 张
```

### 研报简报风格

```markdown
**标题**: [公司名] + [事件] + [评级/观点]
**字数**: 3000-5000 字
**结构**:
  - 投资要点（1 页）
  - 事件概述（0.5 页）
  - 详细分析（3-5 页）
  - 财务预测（1 页）
  - 估值与建议（1 页）
  - 风险提示（0.5 页）
**格式**: Times New Roman、专业客观
**表格**: 3-5 个
**图表**: 8-12 个
```

### 深度分析报告 ⭐ NEW

```markdown
**标题**: [公司名][事件] 深度分析报告
**字数**: 5000-8000 字
**结构**:
  - 执行摘要（300 字）
  - 事件引爆点（核心数据冲击+历史转折点）
  - 战略失误分析（路线错误+时机误判+外部误判）
  - 市场竞争格局（同行业对比+跨行业威胁）
  - 技术路线分析（核心技术竞争力+技术迭代速度）
  - 历史对比分析（行业周期+同类案例）
  - 未来预测模块（短期影响+中长期趋势）
  - 故事化叙事（企业发展史+关键决策分析）
  - 附录（信息来源、数据表格）
**格式**: 专业深度、多维度分析、来源标注
**表格**: 5-8 个
**图表**: 10-15 个
```

## 文件管理规范

### 输出文件夹
- **文件夹名称**: `output`
- **位置**: 项目根目录下 (`/output`)
- **用途**: 仅用于存放生成的文稿，不存放skill文件

### 文件命名规则
- **格式**: `output_YYYYMMDD_HHMMSS.md`
- **示例**: `output_20260321_143000.md`
- **要求**:
  - 日期格式：YYYYMMDD
  - 时间格式：HHMMSS（24小时制）
  - 必须包含日期和时间戳，确保唯一性

### 内容类型命名规则
- **深度分析报告**: `output_YYYYMMDD_HHMMSS.md`
- **B站口播稿**: `output_YYYYMMDD_HHMMSS_bilibili.md`
- **小红书推文**: `output_YYYYMMDD_HHMMSS_xiaohongshu.md`
- **公众号文章**: `output_YYYYMMDD_HHMMSS_wechat.md`
- **研报简报**: `output_YYYYMMDD_HHMMSS_research.md`
- **大纲/脚本**: `output_YYYYMMDD_HHMMSS_outline.md`
