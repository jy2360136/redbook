## 数据采集流程

### Phase 1: 确认事件

```
1. 确认新闻类型
   ├── 财报发布 → 进入财报采集流程
   ├── 产品发布 → 进入产品信息采集流程
   ├── 行业动态 → 进入行业数据采集流程
   └── 政策发布 → 进入政策信息采集流程

2. 确认时效性
   ├── 发布时间是否在72小时内
   ├── 是否已被广泛报道
   └── 是否有更新信息

3. 确认数据完整性
   ├── 核心数据是否齐全
   ├── 是否需要补充数据
   └── 数据来源是否可靠
```

### Phase 2: 核心数据采集

#### 财报类数据清单

```
必须采集：
□ 营业收入（本期、去年同期、同比增长）
□ 毛利润/毛利率
□ 归母净利润
□ 扣非净利润
□ EPS（基本/稀释）
□ 经营性现金流
□ 资产负债表核心项目

建议采集：
□ 分业务收入拆解
□ 分地区收入拆解
□ 费用明细（销售、管理、研发）
□ 主要产品销量/价格
□ 在手订单/合同负债
□ 管理层指引

对比数据：
□ 市场一致预期（营收、利润）
□ 竞品同期数据
□ 历史趋势数据（近8个季度）
```

#### 产品发布数据清单

```
必须采集：
□ 产品名称/型号
□ 发布日期
□ 上市日期/交付日期
□ 价格区间
□ 核心参数（根据产品类型）
□ 目标市场定位

建议采集：
□ 与竞品参数对比
□ 目标市场规模
□ 预期销量/产能
□ 技术亮点
□ 管理层表态

对比数据：
□ 竞品产品参数
□ 历代产品对比
□ 行业平均参数
```

#### 行业动态数据清单

```
必须采集：
□ 事件描述
□ 发生时间
□ 涉及主体
□ 核心影响

建议采集：
□ 历史类似事件
□ 行业数据变化
□ 相关公司表态
□ 专家/机构解读

对比数据：
□ 历史数据对比
□ 同业数据对比
```

#### 政策发布数据清单

```
必须采集：
□ 政策名称
□ 发布机构
□ 发布日期
□ 核心条款
□ 生效时间

建议采集：
□ 政策背景
□ 与征求意见稿差异
□ 专家解读
□ 行业协会表态

对比数据：
□ 历史政策对比
□ 海外类似政策
```

## 数据验证

### 交叉验证原则

```
1. 至少两个独立来源确认
   ✅ 公司公告 + 媒体报道
   ✅ 官方数据 + 第三方数据

2. 数据逻辑检查
   ✅ 增长率计算正确
   ✅ 数据之间逻辑一致
   ✅ 与历史趋势相符

3. 异常数据核实
   ⚠️ 与预期差异>20%的数据需核实
   ⚠️ 与历史趋势明显背离的数据需核实
```

### 常见数据错误

| 错误类型 | 示例 | 检查方法 |
|----------|------|----------|
| 单位错误 | 营收写成"元"而非"亿元" | 与历史数据对比 |
| 计算错误 | 增长率计算失误 | 重新计算 |
| 口径不一 | 营收口径差异 | 确认是否含税 |
| 时间错误 | 季度/年度混淆 | 确认报告期 |

## 数据时效性管理

### 时效性等级

| 等级 | 时间范围 | 处理优先级 |
|------|----------|------------|
| 🔴 极高 | <6小时 | 立即处理 |
| 🟠 高 | <24小时 | 优先处理 |
| 🟡 中 | <72小时 | 正常处理 |
| 🟢 低 | >72小时 | 需重新评估 |

### 数据更新检查

```
1. 财报数据
   - 确认是否为最终版（非预告/快报）
   - 确认是否有后续修正

2. 行业数据
   - 确认数据统计口径
   - 确认是否为最新发布

3. 市场数据
   - 确认数据时间点
   - 注意盘前/盘后差异
```
