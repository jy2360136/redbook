# ASCII 布局设计指南

## 布局原则

### 1. 视口适配（Viewport Fitting）

每张幻灯片必须适配 100vh（视口高度），不能有滚动。

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                     所有内容在一屏内展示                          │
│                                                                 │
│                    不需要滚动即可看完全部内容                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 内容密度控制

| 元素类型 | 最大数量 |
|----------|----------|
| 标题 | 1个 |
| 副标题 | 1个 |
| 要点列表 | 4-6条 |
| 图片 | 1-2张 |
| 数据图表 | 1个 |

### 3. 视觉层次

```
┌─────────────────────────────────────────────────────────────────┐
│  层次1：标题（最大字号）                                          │
│  层次2：副标题/重点数据（中等字号）                                │
│  层次3：正文/要点（标准字号）                                     │
│  层次4：注释/来源（小字号）                                       │
└─────────────────────────────────────────────────────────────────┘
```

## 时长估算

### 估算规则

| 内容类型 | 建议时长 |
|----------|----------|
| 封面/开场 | 5-10秒 |
| 纯文字幻灯片 | 20-40秒/页 |
| 图文幻灯片 | 30-60秒/页 |
| 数据展示幻灯片 | 45-90秒/页 |
| 过渡动画 | 1-2秒 |
| 结尾 | 10-15秒 |

### 计算公式

```
总时长 = Σ(每张幻灯片时长) + 过渡动画时长
```

## ASCII 符号说明

| 符号 | 含义 |
|------|------|
| `┌ ┐ └ ┘ ─ │` | 边框 |
| `├ ┤ ┬ ┴ ┼` | 边框连接 |
| `○ ●` | 时间节点 |
| `→ ← ↑ ↓` | 流程方向 |
| `[ ]` | 占位符 |
| `•` | 要点 |

---

## 注意事项

1. **每张幻灯片必须有唯一标识**：如 `[幻灯片 01]`
2. **时间码必须标注**：如 `00:00-00:10`
3. **配图必须有对应编号**：如 `[img_001.png]`
4. **配乐建议可选**：如 `[配乐建议] 悬疑感背景音乐`
