# 更新日志

## [v1.1.0] - 2026-03-25

### 新增功能
- 新增 `--filter-companies` 开关参数，支持可选的公司名过滤功能
- 创建独立的公司名过滤模块 `company_filter.py`，包含：
  - `filter_by_companies()`: 基础过滤函数
  - `get_company_matches()`: 获取匹配的公司名列表
  - `score_by_companies()`: 公司名匹配打分
  - 预留正则表达式匹配接口

### 改进
- **移除公司名强制过滤**：默认抓取所有新闻，不再受公司名单限制
- **提升抓取效率**：每个源的抓取数量从 30 条提升到 50 条
- **优化输出目录**：统一使用 `news_output_crawl4ai_日期时间` 格式
- **模块化设计**：将公司名过滤逻辑抽离为独立模块，便于后续重构

### 重构
- 删除冗余脚本 `crawl_all_news.py`，整合功能到主脚本
- 所有 `fetch()` 方法增加 `filter_companies` 参数，支持灵活开关过滤
- 公司名单暂时保留在代码中，后续会从配置文件读取

### 修复
- 修复澎湃新闻抓取数量过少的问题（15 条 → 30 条）
- 修复极客公园和晚点 LatePost 抓取数量限制（20 条 → 30 条）

## [v1.0.0] - 2026-03-24

### 新增功能
- 创建了基于Crawl4AI的金融新闻工作流脚本 `financial_news_workflow_crawl4ai.py`
- 实现了 Playwright 浏览器支持，提高抓取能力
- 增加了 20+ 个财经新闻来源，包括国际媒体和国内专业媒体
- 创建了社区论坛抓取工具 `community_crawler.py`，支持雪球和知乎
- 实现了情感分析功能，分析用户评论的情感倾向
- 支持固定输出目录功能，方便结果存储和管理

### 改进
- 移除了财联社的抓取，因为新闻质量差，都是短讯
- 优化了抓取策略，增加了多层备用方案
- 改进了 HTML 解析逻辑，提高内容提取成功率
- 优化了异步处理，提高抓取效率

### 修复
- 修复了异步调用的错误，避免嵌套调用 asyncio.run()
- 修复了 Playwright 浏览器启动的问题

## [v0.1.0] - 2026-03-23

### 初始版本
- 创建了基础的金融新闻工作流脚本
- 实现了 RSS 和 API 源的抓取
- 支持日期过滤和公司名过滤
- 生成小红书风格的分析提示词
