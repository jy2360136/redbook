# 更新日志

## [v1.2.0] - 2026-03-27

### 重大改进

#### 虎嗅网 API 攻克与优化

- **成功破解虎嗅网 API**：
  - 发现并验证了官方 API 端点：`https://api-article.huxiu.com/web/channel/articleListV1`
  - 破解了 POST 参数格式：`platform=www&channel_id=0&last_id={时间戳}`
  - 实现了基于 `last_id` 的分页机制，支持获取约 1200 篇历史文章
  - 成功绕过 WAF 反爬验证，稳定获取文章列表

- **优化发布时间提取**：
  - 从 `https://www.huxiu.com/article/` 列表页直接提取时间
  - 使用 CSS 选择器 `.bottom-line__time` 获取相对时间（如"3 分钟前"）
  - 实现了 Playwright 滚动加载，自动加载更多文章（最多 50 次）
  - 使用单个浏览器实例批量获取，效率提升 10 倍以上

- **优化 JSON 输出格式**：
  - 删除了无用的 `summary` 字段（7 个网站都没有摘要内容）
  - 简化了新闻数据结构，只保留必要字段
  - 发布时间更加准确，使用相对时间格式

### 技术改进

- **批量时间获取策略**：
  - 新增 `batch_fetch_huxiu_times()` 函数
  - 一次性访问列表页，提取所有文章的链接和时间
  - 避免了为每篇文章单独打开详情页的低效做法
  - 实现了 100% 的时间匹配率（1188/1188 篇）

- **代码清理**：
  - 删除了所有测试文件（test\_\*.py）
  - 删除了临时 HTML 和 JSON 文件
  - 删除了废弃的 `get_huxiu_article_time()` 函数
  - 删除了备份文件

### 性能提升

- 虎嗅网抓取数量：23 条 → 1188 条（暴增 52 倍）
- 时间匹配率：0% → 100%
- 发布时间准确性：错误 → 准确（相对时间）
- 整体性能：提升 10 倍以上（批量获取 vs 逐个获取）

### 修复

- 修复了虎嗅网发布时间全部相同的问题
- 修复了 Playwright 逐个访问效率低下的问题
- 修复了 JSON 输出包含无用 summary 字段的问题

## [v1.1.0] - 2026-03-25

### 新增功能

- 新增 `--filter-companies` 开关参数，支持可选的公司名过滤功能
- 创建独立的公司名过滤模块 `company_filter.py`，包含：
  - `filter_by_companies()`: 基础过滤函数
  - `get_company_matches()`: 获取匹配的公司名列表
  - `score_by_companies()`: 公司名匹配打分
  - 预留正则表达式匹配接口

### 改进

- **移除公司名强制过滤**：默认抓取所有新闻，不再受公司名单限制
- **提升抓取效率**：每个源的抓取数量从 30 条提升到 50 条
- **优化输出目录**：统一使用 `news_output_crawl4ai_日期时间` 格式
- **模块化设计**：将公司名过滤逻辑抽离为独立模块，便于后续重构

### 重构

- 删除冗余脚本 `crawl_all_news.py`，整合功能到主脚本
- 所有 `fetch()` 方法增加 `filter_companies` 参数，支持灵活开关过滤
- 公司名单暂时保留在代码中，后续会从配置文件读取

### 修复

- 修复澎湃新闻抓取数量过少的问题（15 条 → 30 条）
- 修复极客公园和晚点 LatePost 抓取数量限制（20 条 → 30 条）
