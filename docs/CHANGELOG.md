# 更新日志

## [v2.3.0] - 2026-03-30

### 重大升级：钛媒体 API + Playwright 混合爬虫

#### 核心改进

**1. 新增 API 直接调用方式** 🚀

- 基于手动抓包结果分析钛媒体官方 API
- API 端点：`https://api.tmtpost.com/v1/lists/new`
- 优势：
  - 更高效：直接获取结构化 JSON 数据
  - 更准确：通过 `item_type` 字段精确过滤视频
  - 更稳定：不依赖网页 DOM 结构
  - 更快速：无需启动浏览器

**2. 视频过滤双重保障** ✅

- **API 级别过滤**：只保留 `item_type == "post"` 的文章
  - 自动排除 `item_type == "video_article"` 的视频内容
- **URL 级别过滤**：检查链接是否包含 `/video/`
- **测试结果**：100% 过滤成功，0 条视频混入

**3. 丰富的元数据** 📊
每篇文章新增字段：

- `author`：作者姓名
- `summary`：文章摘要
- `number_of_reads`：阅读量
- `number_of_upvotes`：点赞数
- `published`：格式化后的发布时间（YYYY-MM-DD HH:MM:SS）

**4. 智能时间过滤优化** ⏰

- 改进时间解析函数，支持更多格式：
  - "约 X 小时以前"、"X 小时前"
  - "约 X 分钟以前"、"X 分钟前"
  - "约 X 天以前"、"X 天前"
  - "上周"、"星期 X"
  - "前天"
  - 绝对日期：YYYY-MM-DD、YYYY.MM.DD
- 优化停止逻辑：
  - 单页所有文章超时时才停止，而非遇到第一篇超时文章就停止
  - 可以获取更多符合时间范围的文章

**5. 目标数量控制** 🎯

- 新增 `--target` 参数，默认 200 条
- 达到目标数量后自动停止
- 支持三种抓取模式：
  - `api`：优先使用 API（默认）
  - `playwright`：仅使用浏览器
  - `both`：API + Playwright 双模式

#### 技术参数

**API 请求头生成**

- 基于真实抓包数据生成
- 包含完整的认证信息：

  - `token`: Base64 编码

- 每页 20 条（limit=20）
- 通过 offset 递增实现分页
- 最多 50 页（约 1000 条）

## [v1.8.0] - 2026-03-29

### 重大改进

#### "查看更多"按钮自动点击功能修复

- **修复 click_count 变量作用域问题**：
  - 问题：`click_count` 变量在 `human_scroll()` 函数的 for 循环内部定义，导致每次循环都重置为 0
  - 修复：将 `click_count = 0` 和 `max_clicks = 6` 移到函数开头，与 `prev_count`、`no_new_count` 同级定义
  - 结果：现在可以正确限制最大点击次数为 6 次，避免无限循环

- **改进点击逻辑**：
  - 新增 `elif click_count >= max_clicks` 分支，当达到最大点击次数时打印提示信息
  - 保留所有原有功能：验证码诊断、3-5 秒等待、人类式滚动
  - 添加注释标记：`# 【新增】自动点击"查看更多"按钮（核心修复）`

- **代码变化**：

  ```python
  # 修复前（错误）：
  for i in range(times):
      click_count = 0  # 每次都重置！
      max_clicks = 6
      if ... and click_count < max_clicks:
          ...

  # 修复后（正确）：
  click_count = 0  # 在函数开头定义
  max_clicks = 6
  for i in range(times):
      if ... and click_count < max_clicks:
          ...
      elif click_count >= max_clicks:
          print("已达到最大点击次数限制")
  ```

### 测试结果

- **之前表现**：成功点击 10 次，加载 330+ 篇文章（30→60→90→...→330）
- **修复后表现**：将限制在 6 次以内，预计加载 210 篇左右（30×7）
- **验证码处理**：每次点击后自动执行 `debug_captcha()` 诊断
- **等待时间**：每次点击后等待 3-5 秒，确保新内容加载完成
