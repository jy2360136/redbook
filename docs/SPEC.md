# 金融新闻自动化工作流需求文档

## 1. 项目概述

### 1.1 项目目标
开发一个自动化的金融新闻工作流，用于：
1. 从专业财经媒体抓取深度报道，确定选题
2. 从社区论坛抓取用户评论和讨论，收集真实评价和细碎信息
3. 生成分析大纲，为内容创作提供基础
4. 支持后续的AI内容生成和视频制作

### 1.2 目标用户
- 财经内容创作者
- 金融分析师
- 投资顾问
- 财经媒体从业者

### 1.3 项目价值
- 提高内容创作效率
- 提供全面的信息来源
- 发现市场热点和舆情
- 辅助投资决策

## 2. 功能需求

### 2.1 专业新闻媒体抓取

#### 2.1.1 支持的媒体源
- **国内专业媒体**：虎嗅网、36氪、第一财经、界面新闻、财新网等
- **国际媒体**：FT中文网、华尔街日报中文版等
- **综合财经媒体**：新浪财经、腾讯财经、网易财经等

#### 2.1.2 抓取功能
- 支持RSS和API源的抓取
- 支持日期范围过滤
- 支持公司名过滤
- 支持多线程/异步抓取，提高效率
- 支持错误处理和重试机制

#### 2.1.3 输出格式
- JSON格式的新闻数据
- 包含标题、来源、链接、摘要、发布时间等信息
- 按相关度排序
- 按公司和来源分组统计

### 2.2 社区论坛信息抓取

#### 2.2.1 支持的社区
- 雪球网
- 知乎

#### 2.2.2 抓取功能
- 支持关键词搜索
- 支持用户评论和讨论的抓取
- 支持情感分析
- 支持多线程/异步抓取

#### 2.2.3 输出格式
- JSON格式的评论数据
- 包含标题、内容、作者、发布时间、点赞数、评论数等信息
- 按情感倾向分组统计
- 按来源分组统计

### 2.3 分析和处理

#### 2.3.1 内容分析
- 新闻相关度计算
- 评论情感分析
- 关键词提取
- 热点话题识别

#### 2.3.2 分析提示词生成
- 生成小红书风格的分析提示词
- 包含事件背景、深度解读、影响分析、投资观点等维度

## 3. 技术需求

### 3.1 技术栈
- Python 3.8+
- asyncio 异步处理
- requests 网络请求
- BeautifulSoup4 HTML解析
- feedparser RSS解析
- Crawl4AI 增强抓取
- Playwright 浏览器自动化

### 3.2 依赖管理
- 使用requirements.txt管理依赖
- 支持离线安装
- 支持版本锁定

### 3.3 性能要求
- 支持并行抓取多个来源
- 抓取速度快，延迟低
- 内存占用合理
- 稳定性高，错误处理完善

## 4. 工作流程

### 4.1 完整工作流
1. **专业新闻搜索**：使用`financial_news_workflow_crawl4ai.py`从专业媒体抓取深度报道，确定选题
2. **社区舆情分析**：使用`community_crawler.py`针对选定的公司/话题，抓取社区讨论和用户评价
3. **大纲生成**：根据收集到的信息，生成包含尽可能多细节的大纲
4. **逐字稿生成**：将大纲输入GPT等工具，生成口播逐字稿
5. **视频生成**：将逐字稿输入AI视频工具，生成5分钟左右的视频

### 4.2 执行顺序
1. 先执行专业新闻抓取，确定核心选题
2. 再执行社区评论抓取，收集用户真实评价
3. 基于两部分信息，进行综合分析
4. 生成内容创作的基础素材

## 5. 部署和运行

### 5.1 环境要求
- Python 3.8+
- pip 20.0+
- 网络连接
- 足够的磁盘空间

### 5.2 安装步骤
1. 克隆项目代码
2. 安装依赖：`pip install -r requirements.txt`
3. 安装Playwright浏览器：`npx playwright install chromium`

### 5.3 运行命令
- **专业新闻抓取**：`python financial_news_workflow_crawl4ai.py --days 10 --sources huxiu,yicai`
- **社区评论抓取**：`python community_crawler.py --keyword "小米汽车" --sources all`

## 6. 验收标准

### 6.1 功能验收
- 能够成功抓取至少10个专业媒体的新闻
- 能够成功抓取雪球和知乎的用户评论
- 能够正确过滤和分析新闻内容
- 能够生成有效的分析提示词

### 6.2 性能验收
- 抓取10个媒体源的时间不超过2分钟
- 抓取100条评论的时间不超过1分钟
- 内存占用不超过500MB
- 错误率不超过10%

### 6.3 输出验收
- 输出格式正确，符合JSON标准
- 包含所有必要的字段
- 数据准确，无明显错误
- 分析结果有参考价值

## 7. 风险和限制

### 7.1 风险
- 网站结构变化导致抓取失败
- 反爬机制导致抓取限制
- 网络不稳定导致抓取超时
- 依赖库版本冲突

### 7.2 限制
- 部分网站需要登录才能访问
- 部分API有访问频率限制
- 抓取内容可能存在版权问题
- 情感分析结果仅供参考，可能存在误差
